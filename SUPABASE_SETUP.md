# ☁️ ربط التطبيق بالإنترنت (Supabase) — دليل خطوة بخطوة
# Connect the app online (Supabase) — step by step

الهدف: حساب واحد يشتغل على **الموقع** وعلى **التطبيق (exe)** بنفس البيانات.
Goal: one account that works on the **website** and the **desktop app**, same data.

> ⏱️ الوقت المتوقع: ~10 دقائق. ما بدها أي برمجة منك.

---

## 1) أنشئ مشروع Supabase مجاني
1. روح على **https://supabase.com** واعمل حساب (مجاني).
2. اضغط **New project** → اختر اسم وكلمة سر لقاعدة البيانات → **Create**.
3. استنى دقيقة لين يجهز المشروع.

## 2) جهّز الجداول (نسخ ولصق مرة وحدة)
1. من القائمة الجانبية: **SQL Editor** → **New query**.
2. افتح الملف **`supabase/schema.sql`** (موجود بمجلد المشروع)، انسخ **كل** محتواه.
3. الصقه بالمحرّر واضغط **Run** ▶️. لازم يطلع «Success».

## 3) أطفئ تأكيد البريد (عشان التسجيل يصير فوري)
1. من القائمة: **Authentication** → **Providers** → **Email**.
2. عطّل خيار **"Confirm email"** ثم **Save**.
   (هيك أي حدا يسجّل بيدخل مباشرة بدون ما ينتظر إيميل تأكيد.)

## 4) خُذ مفتاحَي الربط
1. من القائمة: **Project Settings** (الترس) → **API**.
2. انسخ:
   - **Project URL**  (مثال: `https://abcd1234.supabase.co`)
   - **anon public** key  (مفتاح طويل)

## 5) حطّهم بملف الإعدادات
1. بمجلد المشروع، انسخ الملف **`.env.example`** وسمّي النسخة **`.env`**.
2. عبّيه هيك (بدون مسافات):
```
VITE_SUPABASE_URL=https://abcd1234.supabase.co
VITE_SUPABASE_ANON_KEY=ضع-المفتاح-الطويل-هون
```
3. احفظ الملف.

## 6) جرّب
```
npm run dev
```
- افتح `http://localhost:5173`
- لاحظ إنه فوق صار مكتوب **«متصل / Online»** (بدل «محلي»).
- سجّل عيادة جديدة بإيميل وباسوورد → جرّب تطلع وتفوت من جهاز/متصفّح ثاني بنفس الحساب. 🎉

---

## 7) تنزيل الموقع أونلاين (مجاناً)
أسهل طريقة (Netlify Drop):
1. شغّل: `npm run build`  ← بيعمل مجلد **`dist`**.
2. روح على **https://app.netlify.com/drop** واسحب مجلد **`dist`** كامل.
3. بيعطيك رابط موقع جاهز يفتحه أي حدا ويسجّل فيه.

> ملاحظة: لأن المفاتيح موجودة بملف `.env` وقت ما عملت `build`، الموقع بيكون مربوط بقاعدة بياناتك تلقائياً.

## 8) تحديث تطبيق سطح المكتب (exe) ليصير أونلاين
بعد ما عبّيت `.env`:
```
npm run dist
```
بيعمل exe جديد بمجلد **`release`** مربوط بنفس قاعدة البيانات. وزّعه للناس — أي حساب يفتح على الموقع بيفتح على التطبيق والعكس.

---

## نقاط مهمة
- **الأمان:** كل عيادة بتشوف بيانات مرضاها فقط (مفعّل عبر Row-Level Security بالمخطط).
- **مفتاح anon آمن للنشر** — هو مفتاح عام مصمّم ليكون بالتطبيق، والحماية الحقيقية بقواعد RLS.
- **الوضع المحلي ما بيختفي:** لو شلت ملف `.env`، التطبيق بيرجع يشتغل محلياً (تجريبي) بدون إنترنت.
- **الصور (قبل/بعد، أشعة):** حالياً محفوظة على الجهاز نفسه. مزامنتها عبر السحابة خطوة جاية إذا حبيت.
- **تسجيل دخول منفصل لكل دكتور بالعيادة عبر الإنترنت:** الإصدار الحالي بيدعم حساب صاحب العيادة + ملفات أطباء (ألوان/إسناد). تسجيل دخول مستقل لكل دكتور عبر دعوة = الخطوة الجاية.

أي خطوة وقفت فيها احكيلي وبساعدك. 👌
